Desktop Apps with Node-Webkit

Key Points
  • Download the correct platform binaries from the project site.
  • Set up the standard project structure.
  • Patch up require.js and node.

Explanation

Creating native applications for Windows, Mac And Linux desktop is easy by combining Durandal with Node-Webkit. To get started, visit the Node-Webkit site and download the binaries for the desired platforms you wish to support. Next, create a project structure similar to a typical Durandal web app:

  • App
    • durandal
    • main.js
    • ...your app files here...
  • Content
  • Scripts
  • index.html
  • package.json

The package.json file is used by Node-Webkit. Here's a basic example of what that should look like:

{
  "name": "My App",
  "description": "Description of my app.",
  "version": "1.0",
  "main": "index.html",
  "window": {
    "toolbar": false,
    "width": 1024,
    "height": 768
  }
}

If you want to use the webkit debug tools during development, set toolbar:true. You can find more information on package.json here.

Next, because node has it's own implementation of require that is different from require.js, which Durandal uses, we need to patch things up a bit. To do that, add the following script block to the head of your index.html:

<script type="text/javascript">
    window.gui = require('nw.gui');
    window.requireNode = window.require;
    window.require = undefined;
</script>

This script renames node's require to be requireNode and then globally undefines it so that require.js can use require. Before doing that, we grab the Node-Webkit GUI module and store it globally, since it has some dependencies on node's require implementation.

Finally, you are ready to run your app. To do that you execute nw path/to/app/folder. When you are ready to deploy, check out the Node-Webkit guide on packaging up your app. You will probably want to have a look through their full documentation.

Note: The folder that nw needs is the one containing your index.html file. For simplicity of development, you can also copy the Node-Webkit files into the same folder and just run nw. It will search its folder for a package.json and use that.
